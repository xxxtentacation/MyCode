# 🤖 基于 nanoGPT 的机器人指令语言模型实验报告

## 1. 实验概述

本实验基于 **nanoGPT** 框架，使用 **Studeni/robot-instructions** 数据集，训练一个从自然语言指令到 JSON 机器人命令的字符级语言模型。核心思路是将 NL→JSON 的映射问题转化为序列建模问题，利用 GPT-2 风格的 Transformer 自回归地生成结构化机器人控制命令。

| 项目 | 说明 |
|------|------|
| 基础框架 | nanoGPT (GPT-2 风格 Transformer) |
| 数据集 | Studeni/robot-instructions（NL 指令 → JSON 机器人命令对） |
| 分词策略 | **字符级编码（Character-level Tokenizer）** |
| 模型规模 | 4 层 / 4 头 / 256 维嵌入（约 3.4M 参数） |
| 训练规模 | 3000 iterations，单 GPU |
| 输出目录 | `out-robot-instr/` |

## 2. 数据预处理

### 2.1 数据来源与规模

预处理脚本 `data/robot_instr/prepare.py` 支持两种加载路径：

| 优先级 | 来源 | 说明 |
|--------|------|------|
| 1 | 本地 `input.txt` | 预处理后的格式化文本，**986 条**样本 |
| 2 | HuggingFace 远程 | 通过 `datasets` 库加载 `Studeni/robot-instructions` |

### 2.2 原始数据问题

原始数据存在两类主要问题：

**问题一：移动单位不统一**

数据集中对位置/角度采用了多种不同单位，例如：
- `Move robot TCP to (0.7, 0.4, 0.8) meters`（米）
- `Adjust TCP to position (0.4, 0.6, -0.2) m`（米缩写）
- `Move robot along x, y, and z directions by 1000 mm`（毫米）
- `Rotate the robot's base by 45 degrees`（度）

**问题二：角度小数位数不统一**

角度值存在未约束小数位数的情况，如 `pi/4`（无理数）与 `-30 degrees`（整数）并存。

### 2.3 处理流程

```
原始文本 → 解析为样本对 → 格式化/清洗 → 编码 → train.bin / val.bin / meta.pkl / input.txt
```

- **Step 1 — 数据加载**：优先读取本地 `input.txt`，不存在则从 HuggingFace 远程加载
- **Step 2 — 数据清洗**：`round_floats()` 将所有浮点数统一为 2 位小数；通过 `json.loads` → `json.dumps` 规范化 JSON 输出
- **Step 3 — 格式化**：将每条样本转换为 `Human: <instruction>\nRobot: <json>\n\n` 格式
- **Step 4 — 训练/验证集划分**：按 9:1 顺序切分（**887 train, 99 val**）
- **Step 5 — 编码**：字符级编码，生成 `train.bin`、`val.bin`、`meta.pkl`

### 2.4 输出产物

| 文件 | 格式 | 大小 | 内容 |
|------|------|------|------|
| `train.bin` | `np.uint16` 二进制 | 240 KB | 训练集 token ID 序列 |
| `val.bin` | `np.uint16` 二进制 | 26 KB | 验证集 token ID 序列 |
| `meta.pkl` | Python pickle | 825 B | `vocab_size=77`, `itos`, `stoi` |
| `input.txt` | UTF-8 文本 | 139 KB | 格式化后的完整语料 |

**词汇表概况：**

- **词表大小**：**77** 个唯一字符
- **数据类型**：`uint16`（可表示 0–65535，安全且对齐 nanoGPT 规范）
- **字符集**：包含大小写字母、数字、JSON 语法符号（`{}[]",:`）、小数点、负号、空格换行等

## 3. 分词器：字符级编码

### 3.1 核心实现

```python
# 收集全部唯一字符
chars = sorted(list(set(train_data + val_data)))
vocab_size = len(chars)  # 77

# 建立双向映射
stoi = {ch: i for i, ch in enumerate(chars)}  # 字符 → ID
itos = {i: ch for i, ch in enumerate(chars)}  # ID → 字符

# 编码/解码
def encode(s): return [stoi[c] for c in s]
def decode(ids): return ''.join([itos[i] for i in ids])
```

### 3.2 选择字符级编码的原因

| 考量维度 | 分析 |
|----------|------|
| **词表极小** | 仅 77 个字符，模型参数量极低（嵌入层仅 77×256 参数） |
| **JSON 精确性** | 字符级编码天然精确到每个括号、逗号、数字，适合结构化输出 |
| **无 OOV 问题** | 所有可能字符均在词表中，不存在未登录词 |
| **无需外部分词器** | 不依赖 jieba、tiktoken 等外部库，完全自包含 |
| **与 `shakespeare_char` 一致** | 遵循 nanoGPT 项目的标准字符级做法 |

### 3.3 与其他分词策略的对比

| 分词策略 | 词表大小 | JSON 精度 | 模型复杂度 | 适用场景 |
|----------|----------|-----------|-----------|----------|
| **字符级（本实验）** | 77 | 完美 | 极低 | 小数据集、结构化输出 |
| BPE / tiktoken | 50,257 | 可能歧义 | 高 | 大规模自然语言 |
| Word-level | 数千 | JSON 无法处理 | 高 | 简单分类任务 |
| 空格分隔 | 数百 | 部分丢失 | 中 | 英文任务 |

## 4. 模型架构

### 4.1 整体架构

本实验使用 nanoGPT 的 **GPT-2 风格 Transformer** 模型，核心组件如下：

```
idx (B, T)
   │
   ├── wte: Token Embedding (vocab_size → n_embd)
   ├── wpe: Position Embedding (block_size → n_embd)
   │      │
   │   x = tok_emb + pos_emb  → Dropout
   │      │
   │   Block[0] (Pre-LN + CausalSelfAttention + MLP)
   │   Block[1]
   │   Block[2]
   │   Block[3]               ← 4 层
   │      │
   │   ln_f (LayerNorm)
   │      │
   │   lm_head (n_embd → vocab_size)
   │      │
   ▼   logits + cross_entropy → loss
```

### 4.2 核心组件

| 组件 | 实现 | 说明 |
|------|------|------|
| **LayerNorm** | 自定义，支持 `bias=False` | Pre-LN 归一化，训练更稳定 |
| **CausalSelfAttention** | Flash Attention（PyTorch ≥ 2.0） | 因果自注意力，QKV 合并投影 |
| **MLP** | Linear → GELU → Linear | 4× 扩展比的前馈网络 |
| **Block** | Pre-LN + Attn + MLP + 残差连接 | 标准 Transformer Block |
| **GPT** | 组合所有组件 | 含权重绑定（wte = lm_head） |

### 4.3 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 归一化位置 | **Pre-LN** | 训练更稳定，GPT-2+ 标准 |
| 激活函数 | **GELU** | 与 GPT-2 一致 |
| FFN 扩展比 | **4×** | GPT-2 标准 |
| QKV 投影 | **合并为一个 Linear(3C)** | 减少 kernel launch，提升 GPU 利用率 |
| 注意力实现 | **Flash Attention** | O(N) 内存替代 O(N²)，速度显著提升 |
| 权重绑定 | **wte.weight = lm_head.weight** | 减少参数量 + 正则化效果 |
| 残差初始化 | **c_proj 缩放 1/√(2L)** | 控制深层激活方差 |
| 权重衰减 | **仅 2D+ 参数**（排除 bias/LN） | GPT-2/3 标准做法 |

### 4.4 本实验模型参数

```python
# config/train_robot_instr.py
n_layer = 4       # Transformer 层数
n_head = 4        # 注意力头数
n_embd = 256      # 嵌入维度
block_size = 256  # 最大序列长度（指令+JSON ≈ 100-200 字符，256 足够）
dropout = 0.2     # Dropout 比率（小数据集需要正则化）
bias = False      # 不使用偏置（更快）
```

**参数量估算：**

```
Token Embedding:    77 × 256       ≈ 19,712
Position Embedding: 256 × 256      ≈ 65,536
每个 Block:
  Attn: QKV(256→768) + Proj(256→256) ≈ 262,144
  MLP:  FC(256→1024) + Proj(1024→256) ≈ 524,288
  LN × 2: 2 × 256                     ≈ 512
4 个 Block:                            ≈ 3,147,776
Final LN:                              ≈ 256
lm_head: (与 wte 共享，不额外计数)

总参数 ≈ 3.2M（不含 lm_head）
```

## 5. 训练架构

### 5.1 训练配置

```python
# 核心训练超参数
out_dir = '../out-robot-instr'
dataset = 'robot_instr'
batch_size = 32
block_size = 256
gradient_accumulation_steps = 1

# 优化器
learning_rate = 1e-3  # 初始学习率
min_lr = 1e-4  # 最小学习率（余弦衰减终点）
beta2 = 0.99  # AdamW β₂
weight_decay = 0.1  # 权重衰减（仅对 2D+ 参数）

# 学习率调度
warmup_iters = 50  # 线性 warmup 步数
lr_decay_iters = 3000  # 余弦衰减步数
max_iters = 3000  # 总训练步数

# 评估
eval_interval = 100  # 每 100 步评估一次
eval_iters = 50  # 评估时采样 50 个 batch 取平均
```

### 5.2 训练管线

```
┌──────────────┐    ┌─────────────────┐    ┌──────────┐    ┌──────────────┐
│  data/*.bin  │───▶│  get_batch()    │───▶│  model() │───▶│  loss (CE)   │
│ (np.memmap)  │    │  (随机采样+异步)  │    │  forward │    │              │
└──────────────┘    └─────────────────┘    └──────────┘    └──────┬───────┘
                                                                  │
                                                    ┌─────────────▼───────────┐
                                                    │  optimizer.step()       │
                                                    │  (AdamW + 梯度裁剪)      │
                                                    └─────────────┬───────────┘
                                                                  │
┌──────────────┐    ┌─────────────────┐    ┌──────────┐    ┌──────▼───────┐
│  ckpt.pt     │◀───│  torch.save()   │◀───│  val ↓?  │◀───│  estimate    │
│  metrics.png │    │  (checkpoint)   │    │  (评估)   │    │  _loss()     │
└──────────────┘    └─────────────────┘    └──────────┘    └──────────────┘
```

### 5.3 学习率调度

采用 **Warmup + Cosine Decay** 三段式调度：

```
lr
▲
│   ┌──────────────────┐
│  ╱                    ╲    余弦衰减 (cosine decay)
│ ╱                      ╲──── min_lr = 1e-4
│╱  线性warmup (50步)
└──────────────────────────▶ iter
   0  50              3000
```

### 5.4 训练技巧

| 技巧 | 说明 |
|------|------|
| **混合精度** | `bfloat16` autocast，减少显存占用 |
| **梯度裁剪** | `grad_clip=1.0`，防止梯度爆炸 |
| **异步数据预取** | `pin_memory + non_blocking`，CPU→GPU 传输与计算重叠 |
| **指数平滑 MFU** | EMA(0.9) 平滑 MFU 指标，避免单步波动 |
| **最佳 checkpoint 保存** | 仅 `val_loss` 改善时保存，防止过拟合 |

### 5.5 吞吐量计算

```
tokens_per_iter = gradient_accumulation_steps × batch_size × block_size
                = 1 × 32 × 256 = 8,192 tokens/iter

总训练 tokens = 8,192 × 3,000 = 24,576,000 tokens
```

### 6.5 失败案例分析

10 个未精确匹配的样本主要分为三类错误：

**类型一：数值偏差**（最常见）

模型生成了正确的函数名和 JSON 结构，但具体数值与期望不同：
```
idx=4: joint [1] → [3]  (关节编号偏差)
idx=7: 缺少 z 轴参数 (JSON 结构不完整)
idx=13: z=300.0 → z=350.0 (数值偏差)
```

**类型二：单位换算错误**

```
idx=14: "Move the robotic arm to position (100, -50, 200) cm"
Expected:  {"x": 10000.0, "y": -5000.0, "z": 20000.0}  (cm→mm, ×100)
Generated: {"x": 1000.0, "y": -50.0, "z": 200.0}       (换算不正确)
```

**类型三：多命令拆分/合并错误**

```
idx=0: 3个独立 move_joint → 合并为1个 (joint 合并正确，但 angle[2]=0.01 不正确)
idx=9: 2个关节 → 拆分为3个 move_joint 命令
```

## 7. 架构优劣势分析

### 7.1 训练架构

| 优势 | 可改进点 |
|------|----------|
| 单文件约 460 行，简洁高效 | 直接使用 `np.memmap`，缺乏 DataLoader 抽象 |
| 支持 scratch / resume / GPT-2 三种初始化 | `exec()` 注入 globals() 存在安全隐患 |
| DDP 开箱即用 | 无早停（Early Stopping）机制 |
| 混合精度透明切换 | checkpoint 文件名固定，无版本管理 |
| 优雅中断处理（Ctrl+C 后仍生成图表） | 评估仅为随机采样，非全验证集遍历 |

### 7.2 模型架构

| 优势 | 可改进点 |
|------|----------|
| 完整 GPT 仅 340 行，高度可读 | 仍使用绝对位置编码（无 RoPE/ALiBi） |
| Flash Attention + Fused AdamW + QKV 合并 | 仅支持标准 MHA（无 GQA/MQA） |
| 完善的 HuggingFace 权重迁移 | 激活函数固定为 GELU（不可配置为 SwiGLU） |
| 内置参数量统计 + MFU 估算 | 无 KV Cache，推理效率低 |

## 8. 结论

本实验成功训练了一个 **3.2M 参数**的微型 Transformer 模型，将自然语言机器人指令映射为结构化 JSON 控制命令。核心发现：

1. **字符级编码非常适合结构化输出任务**：77 个字符的极小词表完全覆盖了 JSON 语法和自然语言指令所需的全部字符，JSON 有效率达到 100%

2. **小模型 + 小数据集可行**：仅 887 条训练样本、3000 步训练，即可在 50 条测试集上达到 **80% 精确匹配率**和 **100% 函数名准确率**

3. **主要瓶颈在数值精度**：模型的 JSON 结构生成能力完美，但具体数值（特别是跨单位换算）仍有提升空间

4. **nanoGPT 框架适合快速原型验证**：从数据处理到模型训练、评估、可视化，整个管线简洁高效
