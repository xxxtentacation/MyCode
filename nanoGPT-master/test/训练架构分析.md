# train.py 训练架构分析

## 1. 整体概览

`train.py` 是 nanoGPT 项目的主训练脚本，支持**单GPU调试模式**和**多GPU分布式数据并行（DDP）**两种运行模式。脚本本身是一个扁平结构的自包含训练管线，约460行，无函数/类封装层级。

### 核心设计理念

- **Poor Man's Configurator**：通过 `configurator.py` 将命令行参数直接注入 `globals()`，避免 `config.xxx` 的冗余前缀
- **单一文件自包含**：所有训练逻辑（数据加载、模型初始化、训练循环、评估、日志、可视化）集中在一个文件中
- **兼容多种初始化方式**：`scratch`（从零训练）、`resume`（断点续训）、`gpt2*`（从OpenAI GPT-2权重初始化）

## 3. 架构分层详解

### 3.1 第一层：配置系统（L35–83）

```
┌─────────────────────────────────────────────────────┐
│                   Configuration Layer               │
├─────────────────────────────────────────────────────┤
│  硬编码默认值（Python 变量）                           │
│       │                                              │
│       ▼                                              │
│  configurator.py 覆盖（CLI --key=value 或配置文件）     │
│       │                                              │
│       ▼                                              │
│  收集到 config dict（用于日志 / checkpoint）           │
└─────────────────────────────────────────────────────┘
```

**关键配置项分类：**

| 类别 | 变量 | 说明 |
|------|------|------|
| I/O | `out_dir`, `eval_interval`, `log_interval`, `eval_iters` | 输出目录与评估频率 |
| 数据 | `dataset`, `batch_size`, `block_size`, `gradient_accumulation_steps` | 数据集与批次参数 |
| 模型 | `n_layer`, `n_head`, `n_embd`, `dropout`, `bias` | GPT 架构超参数 |
| 优化器 | `learning_rate`, `weight_decay`, `beta1`, `beta2`, `grad_clip` | AdamW 参数 |
| 学习率 | `decay_lr`, `warmup_iters`, `lr_decay_iters`, `min_lr` | 余弦衰减策略 |
| 分布式 | `backend` | DDP 通信后端（nccl/gloo） |
| 系统 | `device`, `dtype`, `compile` | 硬件/精度/编译选项 |
| 日志 | `wandb_log`, `wandb_project`, `wandb_run_name` | W&B 实验追踪 |

### 3.2 第二层：运行环境初始化（L86–117）

```
ddp 检测 (RANK 环境变量)
    │
    ├── DDP 模式:
    │   ├── init_process_group(backend)
    │   ├── 提取 ddp_rank / ddp_local_rank / ddp_world_size
    │   ├── 锁定 CUDA device
    │   ├── master_process = (rank==0)
    │   └── 梯度累积步数 ÷= world_size  （保证等效 batch size 一致）
    │
    └── 单机模式:
        ├── master_process = True
        └── ddp_world_size = 1

随机种子 = 1337 + seed_offset  （DDP 下每个进程种子不同）
启用 tf32 加速 (matmul + cudnn)
自动混合精度 context 创建（bfloat16 / float16 / float32）
```

**设计亮点：**
- `ddp_world_size` 分摊梯度累积步数：多卡训练时等效 batch size 不变，用户无需手动调整
- `master_process` 标志位控制日志/checkpoint/summary 只在 rank=0 进程执行

### 3.3 第三层：数据加载（L119–136）

```python
get_batch(split):
    1. 根据 split 选择 train.bin 或 val.bin
    2. 使用 np.memmap 创建内存映射（uint16，只读模式）
    3. 随机采样 block_size 长度的连续序列
    4. x = data[i : i+block_size]      # 输入序列
       y = data[i+1 : i+1+block_size]  # 目标序列（shift 1）
    5. CUDA 下 pin_memory + non_blocking 异步传输
    6. 返回 (x, y)
```

**设计亮点：**
- **每次调用重新创建 memmap**：修复 NumPy 的已知内存泄漏问题
- **Pin Memory 异步传输**：将 CPU→GPU 的数据搬运与 GPU 前向计算重叠，提升吞吐
- **uint16 存储**：适合小词表场景，节省磁盘空间

### 3.4 第四层：模型初始化（L138–198）

```
          ┌──────────────────┐
          │  模型初始化策略    │
          └────┬─────┬───────┘
               │     │
     ┌─────────▼┐  ┌─▼──────────────┐  ┌─▼────────────────┐
     │ scratch  │  │ resume         │  │ gpt2*            │
     │ 新建GPT  │  │ 从ckpt.pt恢复  │  │ from_pretrained  │
     │          │  │ + optimizer    │  │ 加载预训练权重    │
     └──────────┘  └────────────────┘  └──────────────────┘
```

**关键流程：**
1. 尝试从 `meta.pkl` 读取词表大小
2. 根据 `init_from` 选择初始化路径
3. 若 `block_size < model.config.block_size`，对模型做 **model surgery** 裁剪
4. 将模型移到目标 device

**模型核心依赖：**
- `model.py::GPTConfig` — 配置数据类
- `model.py::GPT` — GPT-2 风格的 Transformer 模型

### 3.5 第五层：优化器与编译（L200–219）

```
GradScaler (float16 时启用)
    ↓
model.configure_optimizers(weight_decay, lr, betas, device_type)
    → 返回 AdamW optimizer（模型内部方法，支持分组权重衰减）
    ↓
torch.compile(model)  [if compile=True]
    → PyTorch 2.0 JIT 编译，约 1 分钟预热，推理加速显著
    ↓
DDP(model)  [if ddp=True]
    → 包裹模型为分布式数据并行
```

**注意执行顺序**：先 compile，后 DDP — 这样每个 rank 独立编译，避免 DDP 包装带来的复杂性。

### 3.6 第六层：训练循环（L269–368）—— 核心

```
┌──────────────────────────────────────────────────────────┐
│                    Training Loop                          │
│                                                          │
│  while True:                                             │
│    │                                                     │
│    ├─ [1] 学习率调度  get_lr(iter_num)                    │
│    │      ├── warmup: 线性增长                             │
│    │      ├── cosine decay: 余弦衰减                      │
│    │      └── min_lr: 下限截断                             │
│    │                                                     │
│    ├─ [2] 定期评估  (iter % eval_interval == 0)           │
│    │      ├── estimate_loss()  在 train/val 上计算 loss   │
│    │      ├── 记录 metrics                               │
│    │      ├── wandb 日志 (可选)                           │
│    │      └── 若 val_loss 改善 → 保存 checkpoint          │
│    │                                                     │
│    ├─ [3] 梯度累积循环  (micro_step: 0..ga_steps)         │
│    │      ├── DDP: 仅最后一步同步梯度                      │
│    │      ├── autocast 前向计算 → loss                    │
│    │      ├── scaler.scale(loss).backward()              │
│    │      └── 异步预取下一 batch                          │
│    │                                                     │
│    ├─ [4] 梯度裁剪  (grad_clip)                           │
│    │      scaler.unscale_() → clip_grad_norm_()          │
│    │                                                     │
│    ├─ [5] 优化器更新  scaler.step() + scaler.update()     │
│    │      optimizer.zero_grad(set_to_none=True)          │
│    │                                                     │
│    ├─ [6] 日志记录  MFU / loss / dt                       │
│    │                                                     │
│    ├─ [7] iter_num++ → 检查终止条件                       │
│    │                                                     │
│    └─ KeyboardInterrupt: 优雅退出，仍生成图表               │
└──────────────────────────────────────────────────────────┘
```

#### 3.6.1 `estimate_loss()` — 损失评估函数

```python
@torch.no_grad()
def estimate_loss():
    model.eval()
    for split in ['train', 'val']:
        losses = torch.zeros(eval_iters)
        for k in range(eval_iters):    # 采样 eval_iters 个 batch
            X, Y = get_batch(split)
            logits, loss = model(X, Y)
            losses[k] = loss.item()
        out[split] = losses.mean()     # 取平均，减少采样误差
    model.train()
    return out
```

评估 `eval_iters` 个 batch 取平均，兼顾准确性和速度。

#### 3.6.2 `get_lr()` — 学习率调度策略

采用 **Warmup + Cosine Decay** 三段式调度：

```
lr
▲
│   ┌──────────────────┐
│  ╱                    ╲    余弦衰减
│ ╱                      ╲──── min_lr
│╱  线性warmup
└──────────────────────────▶ iter
   warmup_iters  lr_decay_iters
```

#### 3.6.3 梯度累积 + 数据预取

梯度累积允许在小 GPU 上模拟大 batch：

```
total_batch_size = batch_size × block_size × gradient_accumulation_steps × ddp_world_size
```

**异步预取技巧**：紧接在 forward 之后立即发起下一个 batch 的 I/O，与 GPU 计算重叠。

#### 3.6.4 混合精度训练

通过 `torch.amp.autocast` + `torch.amp.GradScaler` 实现自动混合精度：

| 数据类型 | autocast | GradScaler | 说明 |
|----------|----------|------------|------|
| float32 | ❌ | ❌ | 纯 FP32 训练 |
| bfloat16 | ✅ forward 中自动降低到 BF16 | ❌ | BF16 动态范围大，无需 scaler |
| float16 | ✅ forward 中自动降低到 FP16 | ✅ | FP16 需要梯度缩放防止下溢 |

### 3.7 第七层：结果可视化（L370–459）

## 5. 关键设计模式与技巧

### 5.1 DDP 梯度同步控制

```python
model.require_backward_grad_sync = (micro_step == gradient_accumulation_steps - 1)
```
在梯度累积中，只在最后一个 micro_step 触发 all-reduce 梯度同步，避免不必要的通信开销。

### 5.2 权重衰减分组

模型通过 `configure_optimizers()` 内部方法将参数分为两组：
- **需要衰减**：权重矩阵（weight）
- **不需要衰减**：偏置（bias）和 LayerNorm 参数

### 5.3 Checkpoint 结构

```python
checkpoint = {
    'model': raw_model.state_dict(),      # 模型权重（去掉 DDP 包装和 compile 前缀）
    'optimizer': optimizer.state_dict(),   # 优化器状态
    'model_args': model_args,              # 模型架构参数
    'iter_num': iter_num,                  # 训练步数
    'best_val_loss': best_val_loss,        # 最佳验证损失
    'config': config,                      # 完整配置快照
}
```

### 5.4 compile 兼容性处理

```python
# 移除 torch.compile 添加的 _orig_mod. 前缀
unwanted_prefix = '_orig_mod.'
for k,v in list(state_dict.items()):
    if k.startswith(unwanted_prefix):
        state_dict[k[len(unwanted_prefix):]] = state_dict.pop(k)
```

### 5.5 MFU 指数平滑

```python
running_mfu = mfu if running_mfu == -1.0 else 0.9 * running_mfu + 0.1 * mfu
```
使用指数移动平均（EMA）平滑 MFU 指标，避免单步波动。

### 5.6 eval_only 模式

`iter_num == 0 and eval_only` 时立即退出 — 用于快速验证数据加载和模型初始化是否正确。

## 8. 架构优劣势分析

### 优势
1. **简洁高效**：单文件约 460 行，无框架依赖，适合教学和快速迭代
2. **灵活初始化**：支持 scratch / resume / GPT-2 预训练三种路径
3. **DDP 开箱即用**：一行 `torchrun` 即可多卡训练
4. **混合精度透明**：自动根据硬件能力和 dtype 配置决定是否启用 GradScaler
5. **完善的监控**：内置 metrics 记录、wandb 集成、训练后图表生成
6. **优雅中断处理**：Ctrl+C 后仍生成图表和保存指标

### 可改进点
1. **无 DataLoader 抽象**：直接使用 `np.memmap` + 随机采样，缺乏 shuffle / 多 worker 加载
2. **配置管理简陋**：`exec()` 注入 globals() 存在安全隐患，且不支持类型校验
3. **无验证集独立评估**：`estimate_loss` 使用的是训练中的随机采样，不是遍历全验证集
4. **硬编码较多**：图表布局、matplotlib 样式等直接写死在代码中
5. **无早停（Early Stopping）**：仅靠 `max_iters` 终止，无 patience 机制
6. **无实验管理**：checkpoint 文件名固定为 `ckpt.pt`，无时间戳或多版本管理
