# model.py 架构分析

## 1. 整体概览

`model.py` 是 nanoGPT 的核心模型文件，完整实现了 GPT-2 风格的 Transformer 语言模型，全部集中在单个文件中（约 340 行）。参照了两个权威实现：
- OpenAI 官方 GPT-2 TensorFlow 实现
- HuggingFace Transformers PyTorch 实现

## 2. 组件总览

```
model.py
│
├── LayerNorm              (18-27)   带可选 bias 的 LayerNorm
├── CausalSelfAttention    (29-76)   因果自注意力（Flash / 手动）
├── MLP                    (78-92)   FFN 前馈网络（4x 扩展 + GELU）
├── Block                  (94-106)  Transformer Block（Pre-LN + 残差）
├── GPTConfig              (108-116) 配置 dataclass
└── GPT                    (118-341) 主模型类
    ├── __init__()                  初始化 + 权重绑定 + 参数初始化
    ├── get_num_params()            参数统计
    ├── _init_weights()             权重初始化策略
    ├── forward()                   前向传播（训练/推理）
    ├── crop_block_size()           模型手术：裁剪 block_size
    ├── from_pretrained()           从 HuggingFace GPT-2 加载权重
    ├── configure_optimizers()      构建分组建 AdamW 优化器
    ├── estimate_mfu()              模型 FLOPs 利用率估算
    └── generate()                  自回归文本生成
```

## 3. 组件层级详解（自底向上）

### 3.1 LayerNorm — 可选偏置的层归一化

```
输入: (B, T, C)
配置: ndim=C, bias=bool
输出: (B, T, C) — 归一化后
```

```
python
class LayerNorm(nn.Module):
    weight: nn.Parameter(ones)    # γ, 可学习缩放
    bias:   nn.Parameter(zeros) | None  # β, 可选偏置
    forward F.layer_norm(input, eps=1e-5)
```

**设计要点：** PyTorch 原生 `nn.LayerNorm` 不支持 `bias=False`，此类提供该功能。GPT-2 原始实现中 bias=True，但 nanoGPT 默认 bias=False（更快且效果略好）。

### 3.2 CausalSelfAttention — 因果自注意力

```
输入: x (B, T, C)
       │
       ▼
  c_attn: Linear(C → 3C)        # QKV 合并投影，一次矩阵乘法
       │
       ├── split → q, k, v       # 沿通道维切分为三份
       │
       ├── view + transpose      # (B,T,C) → (B,nh,T,hs)
       │
       ├── 注意力计算 [分支]:
       │   ┌─ flash=True:  scaled_dot_product_attention(q,k,v, is_causal=True)
       │   └─ flash=False: Q@K^T / √d → masked_fill → softmax → dropout → @V
       │
       ├── transpose + view      # 合并多头 (B,nh,T,hs) → (B,T,C)
       │
       ▼
  c_proj: Linear(C → C)          # 输出投影
  resid_dropout
       │
       ▼
输出: (B, T, C)
```

#### 两种注意力实现对比

| 特性 | Flash Attention (PyTorch ≥ 2.0) | 手动实现 |
|------|-------------------------------|---------|
| 函数 | `F.scaled_dot_product_attention` | QK^T / softmax / dropout 手动步骤 |
| 因果掩码 | `is_causal=True` 参数 | `register_buffer("bias")` 下三角矩阵 |
| 内存 | O(N) 而非 O(N²)，融合 kernel | 需显式存储注意力矩阵 (B,nh,T,T) |
| 速度 | 显著更快 | 基准实现，兼容旧 PyTorch |
| dropout | 原生支持 | 手动 `attn_dropout` |

#### QKV 合并投影设计

传统实现用三个独立 Linear，nanoGPT 用一个 `Linear(C, 3C)` 同时计算 Q、K、V，然后沿维度切分。这样做：
- **减少 kernel launch 次数**（1 次大矩阵乘法 vs 3 次小矩阵乘法）
- **更好的 GPU 利用率**

### 3.3 MLP — 前馈网络

```
输入: x (B, T, C)
       │
       ▼
  c_fc: Linear(C → 4C)          # 扩展 4 倍
  GELU()                         # 激活函数
  c_proj: Linear(4C → C)        # 投影回原维度
  Dropout
       │
       ▼
输出: (B, T, C)
```

遵循 GPT-2 标准：**4× 扩展比 + GELU 激活**。与一些现代模型使用 SwiGLU 不同，这里保持与原始 GPT-2 一致。

### 3.4 Block — Transformer 块

```
输入: x (B, T, C)
       │
       ├──→ ln_1 → CausalSelfAttention → (+) ──┐
       │                    ↑                    │
       │              (残差连接)                  │
       │                                         │
       ├──→ ln_2 → MLP → (+) ──────────────────┤
       │              ↑                          │
       │        (残差连接)                        │
       ▼                                         │
输出: x (B, T, C)
```

**架构选择：Pre-LN（前置层归一化）**

```
Pre-LN:   x = x + Attn(LN(x))
          x = x + MLP(LN(x))

Post-LN:  x = LN(x + Attn(x))    ← 原始 Transformer / GPT-1 使用
          x = LN(x + MLP(x))
```

Pre-LN 的优势：
- **训练更稳定**：梯度不需要穿过 LayerNorm 传播
- **对学习率不敏感**：不需要 warmup 也能收敛
- 这是 GPT-2 及之后所有 GPT 系列的标准做法

### 3.5 GPTConfig — 配置数据类

```python
@dataclass
class GPTConfig:
    block_size: int = 1024       # 最大序列长度
    vocab_size: int = 50304      # 词表大小（50257 ↑ 64 的倍数）
    n_layer: int = 12            # Transformer 层数
    n_head: int = 12             # 注意力头数
    n_embd: int = 768            # 嵌入维度
    dropout: float = 0.0         # Dropout 比率
    bias: bool = True            # 是否在 Linear / LayerNorm 中使用 bias
```

**vocab_size 为何是 50304？** GPT-2 真实词表大小为 50257，向上取整到 64 的倍数（50304）以提高 GPU 计算效率 — 多余的 token 永远不会被预测，但不影响训练。

### 3.6 GPT — 主模型（核心）

#### 3.6.1 模型结构图

```
                  idx (B, T)
                     │
          ┌──────────┼──────────┐
          ▼                     ▼
    wte: Embedding        wpe: Embedding
    (vocab_size→n_embd)   (block_size→n_embd)
          │                     │
          └──────────┬──────────┘
                     ▼
                tok_emb + pos_emb
                     │
                     ▼
                 Dropout
                     │
          ┌──────────┼──────────┐
          ▼                     ▼
    Block[0] (Pre-LN + Attn + MLP)
          │
          ▼
    Block[1]
          │
          ▼
         ...
          │
          ▼
    Block[n_layer-1]
          │
          ▼
        ln_f (LayerNorm)
          │
          ├── targets is not None (训练)
          │       ▼
          │   lm_head: Linear(n_embd → vocab_size)
          │       │
          │       ▼
          │   logits (B,T,vocab_size), loss = cross_entropy(logits, targets)
          │
          └── targets is None (推理)
                  ▼
              lm_head(x[:,[-1],:])   # 仅计算最后一个位置的 logits
                  │
                  ▼
              logits (B,1,vocab_size), loss = None
```

#### 3.6.2 关键设计决策

**1. Weight Tying（权重绑定）**

```python
self.transformer.wte.weight = self.lm_head.weight
```

将 **输入词嵌入矩阵** 与 **输出投影矩阵** 共享权重，这是 GPT-2 论文中使用的技术。好处：
- 减少参数量（对于 GPT-2 124M 约减少 38M 参数）
- 作为正则化手段，略微提升泛化性能

由于 wte 和 lm_head 共享权重，`get_num_params()` 中 `non_embedding=True` 时只减去位置编码参数而保留 token 嵌入（因为它们同时服务于输出层）。

**2. 权重初始化策略**

```python
# 通用初始化
Linear.weight   → N(0, 0.02)
Linear.bias     → zeros
Embedding       → N(0, 0.02)

# 残差投影特殊缩放（GPT-2 论文技巧）
c_proj.weight   → N(0, 0.02 / √(2 × n_layer))
```

残差投影（`c_proj`）的初始化标准差随层数缩小，确保深层网络的激活方差不会随深度增加而爆炸 — 这是 GPT-2 论文的关键初始化技巧。

**3. 推理时的优化**

```python
if targets is None:
    logits = self.lm_head(x[:, [-1], :])  # 仅最后一个 token
```

训练时需要所有位置的 logits 计算 loss，但推理时只需要最后一个 token 的预测 — 用 `[-1]` 而非 `-1` 保持时间维度（shape 为 `(B,1,C)` 而非 `(B,C)`），避免维度不一致。

#### 3.6.3 方法详解

---

##### `get_num_params(non_embedding=True)`

统计模型参数数量。`non_embedding=True` 时减去位置编码参数（但保留 token 嵌入，因其同时用于 lm_head）。

---

##### `forward(idx, targets=None)`

**输入：**
- `idx`: (B, T) 的 token 索引序列
- `targets`: (B, T) 的目标序列，为 `None` 时进入推理模式

**流程：**
1. 构建位置编码 `torch.arange(0, t)`
2. token 嵌入 + 位置嵌入 → Dropout
3. 逐层通过 n_layer 个 Transformer Block
4. 最终 LayerNorm
5. 训练模式：完整的 lm_head → cross_entropy loss
6. 推理模式：仅最后一个位置的 lm_head → 返回 logits

**输出：** `(logits, loss)` — 推理时 loss 为 None

---

##### `crop_block_size(block_size)`

"模型手术"：在不重新训练的情况下减小 block_size。

```python
# 步骤：
1. 裁剪位置嵌入矩阵 → wpe.weight[:block_size]
2. 更新 config.block_size
3. 循环裁剪所有 Block 中 attention 的 bias 矩阵（如果存在）
```

典型使用场景：从 GPT-2（block_size=1024）加载权重，但只在小数据集上训练短序列。

---

##### `from_pretrained(model_type, override_args=None)` — 类方法

从 HuggingFace Transformers 加载 OpenAI GPT-2 官方权重。

```
流程:
  1. 根据 model_type 确定架构参数
     gpt2         → 12L/12H/768d  (124M)
     gpt2-medium  → 24L/16H/1024d (350M)
     gpt2-large   → 36L/20H/1280d (774M)
     gpt2-xl      → 48L/25H/1600d (1558M)

  2. 创建 nanoGPT 模型实例 + HuggingFace 模型实例
  3. 逐参数对齐并复制权重
     ├── 普通参数：直接复制
     └── Conv1D 参数 (c_attn, c_proj, c_fc)：需要转置
         (OpenAI 使用 Conv1D，nanoGPT 使用 nn.Linear)
  4. 忽略非参数 buffer (attn.bias, attn.masked_bias)
```

**Conv1D → Linear 转置适配：**

OpenAI 原始实现使用 `Conv1D`（权重 shape 为 `[in_features, out_features]`），而 PyTorch 的 `nn.Linear` 使用 `[out_features, in_features]`。加载时需要将所有 projection 层权重转置：

```python
transposed = ['attn.c_attn.weight', 'attn.c_proj.weight',
              'mlp.c_fc.weight', 'mlp.c_proj.weight']
# sd[k].copy_(sd_hf[k].t())  ← 转置复制
```

---

##### `configure_optimizers(weight_decay, learning_rate, betas, device_type)`

构建 **分组权重衰减** 的 AdamW 优化器。

```
参数分组策略：
  ┌──────────────────────────────────────┐
  │  Group 1: 2D+ 参数 → weight_decay    │
  │  - Linear.weight (矩阵乘法的权重)      │
  │  - Embedding.weight                  │
  │  - LayerNorm.weight (2D 视为张量)     │
  ├──────────────────────────────────────┤
  │  Group 2: <2D 参数 → weight_decay=0  │
  │  - Linear.bias                       │
  │  - LayerNorm.bias                    │
  └──────────────────────────────────────┘
```

**分组逻辑：** `p.dim() >= 2` 的参数施加权重衰减，`p.dim() < 2` 的参数（偏置和 LayerNorm 权重虽为 1D）不衰减。这是 GPT-2/3 论文中的标准做法。

**Fused AdamW：** CUDA 环境下自动检测并启用 PyTorch 的融合 AdamW kernel，减少 kernel launch 开销。

---

##### `estimate_mfu(fwdbwd_per_iter, dt)`

估算 **Model FLOPs Utilization (MFU)**，以 A100 bfloat16 峰值算力为基准。

```
参考: PaLM 论文 Appendix B

每 token 前向 FLOPs ≈ 6N + 12 × L × H × Q × T
  6N:       嵌入和输出层的 FLOPs（N = 非嵌入参数数）
  12L·H·Q·T: 注意力计算 FLOPs（Q = head_dim = n_embd/n_head）

整个 batch 前向+反向:
  flops_per_iter = flops_per_token × block_size × fwdbwd_per_iter

MFU = (flops_per_iter / dt) / 312e12
      实际吞吐量          A100 bf16 峰值
```

**注意：** MFU 是近似估算，仅考虑矩阵乘法和注意力操作，忽略了 LayerNorm、激活函数等次要计算。

---

##### `generate(idx, max_new_tokens, temperature=1.0, top_k=None)`

自回归文本生成，支持三种解码策略：

```
for _ in range(max_new_tokens):
    1. 裁剪上下文至 block_size
    2. 前向传播获取 logits
    3. 取最后一个位置的 logits
    4. 解码策略:
       ├── temperature=0: 贪心解码 (argmax)
       └── temperature>0:
           ├── logits / temperature (温度缩放)
           ├── top_k 过滤 (可选)
           ├── softmax → 概率分布
           └── multinomial 采样
    5. 将新 token 拼接到序列末尾
```

**温度参数效果：**
- `temperature → 0`：确定性输出，总是选最高概率 token
- `temperature = 1.0`：按原始 softmax 概率采样
- `temperature → ∞`：趋近均匀分布

**top_k 过滤：** 仅保留概率最高的 k 个 token，将其他 token 概率置为 `-inf`，防止低概率 token 被采样。

## 4. 完整数据流（训练模式）

```
idx (B,T)  targets (B,T)
  │
  ├──→ wte(idx) ──→ tok_emb (B,T,C)
  │                      │
  ├──→ wpe(pos) ──→ pos_emb (T,C)
  │                      │
  │         x = tok_emb + pos_emb
  │              │
  │         Dropout
  │              │
  │    ┌─────────┴──────────┐
  │    │   for block in h:  │  × n_layer
  │    │     x = x + Attn(LN1(x))
  │    │     x = x + MLP(LN2(x))
  │    └─────────┴──────────┘
  │              │
  │         ln_f(x)
  │              │
  │         lm_head(x) ──────────→ logits (B,T,vocab_size)
  │                                      │
  └──────────────────────────────────────┤
                                    cross_entropy(
                                      logits.view(-1, vocab_size),
                                      targets.view(-1),
                                      ignore_index=-1
                                    )
                                         │
                                         ▼
                                      loss (scalar)
```

## 5. 架构设计决策总结

| 决策 | 选择 | 理由 |
|------|------|------|
| 归一化位置 | Pre-LN | 训练更稳定，GPT-2+ 标准 |
| 激活函数 | GELU | 与 GPT-2 一致 |
| FFN 扩展比 | 4× | GPT-2 标准 |
| QKV 投影 | 合并为一个 Linear(3C) | 减少 kernel launch |
| 注意力实现 | Flash Attention (PyTorch 2.0+) | 内存与速度优势 |
| 权重绑定 | wte.weight = lm_head.weight | 减少参数 + 正则化 |
| 残差初始化 | c_proj 缩放 1/√(2L) | 控制深层激活方差 |
| 偏置 | 默认 False | 比 GPT-2 的 True 略快且效果更好 |
| 权重衰减 | 仅 2D+ 参数（排除 bias/LN） | GPT-2/3 标准 |
| 位置编码 | 可学习绝对位置编码 | 与 GPT-2 一致 |

## 6. 参数量估算

对于 GPT-2 标准配置 (12L/12H/768d)：

```
Token Embedding:     50257 × 768  ≈ 38.6M
Position Embedding:  1024 × 768   ≈ 0.79M
每个 Block:
  Attn: QKV(768→2304) + Proj(768→768) ≈ 2.36M
  MLP:  FC(768→3072) + Proj(3072→768) ≈ 4.72M
  LN × 2: 2 × 768 × 2                ≈ 0.003M
12 个 Block:                           ≈ 85.1M
Final LN:                              ≈ 0.001M
lm_head: (与 wte 共享，不额外计数)

总参数 ≈ 124.4M  (non_embedding: 85.0M)
```

## 9. 代码质量评价

### 优势
1. **极简实现**：完整 GPT 模型仅 340 行，高度可读
2. **性能优先**：Flash Attention、Fused AdamW、QKV 合并投影
3. **兼容性好**：自动检测 PyTorch 版本并回退到手动注意力
4. **权重迁移**：完善的 HuggingFace → nanoGPT 权重转换
5. **实用工具**：内置参数量统计、MFU 估算、文本生成

### 可改进点
1. **无 RoPE/ALiBi**：仍使用绝对位置编码，非当前 SOTA
2. **无 GQA/MQA**：仅支持标准多头注意力（Multi-Head Attention）
3. **激活函数固定**：始终使用 GELU，不可配置为 SwiGLU 等变体
4. **无 KV Cache**：生成时每次重新计算全部历史，推理效率低
5. **ModuleDict 包装**：`self.transformer.xxx` 的嵌套访问略显冗余
6. **compile 兼容性警告**：weight tying + torch.compile 存在未解决的 UserWarning
